ЧАСТЬ 1: ВВЕДЕНИЕ И БЫСТРЫЙ СТАРТ

1.1. О СИСТЕМЕ
───────────────────────────────────────────────────────────────────────────────

Система бухгалтерского учёта - это полностью готовое WPF приложение для ведения
бухгалтерского учёта организации. Приложение разработано специально для 
демонстрационного экзамена БОМ 26.

Основные возможности:
• Управление справочниками (контрагенты, товары, счета)
• Создание первичных документов
• Ведение бухгалтерских проводок по принципу двойной записи
• Формирование оборотно-сальдовой ведомости
• Экспорт отчётов в CSV/Excel
• Автоматическая проверка правильности учёта

Технические характеристики:
• Платформа: .NET 6.0
• Язык программирования: C#
• Технология UI: WPF (Windows Presentation Foundation)
• База данных: SQLite 3 (встроенная, не требует установки сервера)
• Кодировка: UTF-8
• Язык интерфейса: Русский

Преимущества:
✓ Не требует установки MySQL или других серверов БД
✓ База данных создаётся автоматически при первом запуске
✓ Все данные хранятся локально в одном файле
✓ Простой запуск через один BAT файл
✓ Полная документация на русском языке

1.4. О БАЗЕ ДАННЫХ
───────────────────────────────────────────────────────────────────────────────

Тип базы данных: SQLite (встроенная)
Файл: accounting.db
Расположение: DatabaseApp\DatabaseApp\bin\Release\net6.0-windows\

База данных создаётся АВТОМАТИЧЕСКИ при первом запуске приложения!

Что уже есть в базе данных:
✓ 2 пользователя (admin, user) с паролями
✓ 29 счетов бухгалтерского учёта (полный план счетов)
✓ 10 таблиц с полной структурой
✓ Индексы для быстрого поиска
✓ Связи между таблицами (внешние ключи)

Таблицы базы данных:
1. users - Пользователи системы
2. organizations - Организации
3. counterparties - Контрагенты (поставщики и покупатели)
4. accounts - План счетов бухгалтерского учёта
5. documents - Первичные документы
6. transactions - Бухгалтерские проводки
7. products - Товары и услуги
8. inventory_operations - Операции с товарами
9. bank_accounts - Банковские счета
10. cash_operations - Кассовые операции

Что НЕ нужно делать:
✗ Устанавливать MySQL или PostgreSQL
✗ Настраивать сервер базы данных
✗ Создавать таблицы вручную
✗ Импортировать SQL скрипты
✗ Настраивать подключение к БД

Всё уже готово! Просто запустите приложение.

2.2. УСТАНОВКА .NET SDK
───────────────────────────────────────────────────────────────────────────────

.NET SDK - это обязательный компонент для работы приложения!

Шаг 1: Скачивание
   1. Откройте браузер
   2. Перейдите на сайт: https://dotnet.microsoft.com/download/dotnet/6.0
   3. Найдите раздел ".NET 6.0 SDK"
   4. Нажмите кнопку "Download .NET 6.0 SDK (x64)" для Windows
   5. Дождитесь завершения загрузки

Шаг 2: Установка
   1. Найдите скачанный файл (обычно в папке "Загрузки")
   2. Двойной клик по файлу установки
   3. Нажмите "Install" (Установить)
   4. Дождитесь завершения установки (2-5 минут)
   5. Нажмите "Close" (Закрыть)

Шаг 3: Проверка установки
   1. Откройте командную строку (Win+R, введите cmd, нажмите Enter)
   2. Введите команду: dotnet --version
   3. Нажмите Enter
   4. Должна отобразиться версия .NET SDK (например: 6.0.100)

Если версия отображается - установка прошла успешно!
Если появляется ошибка - перезагрузите компьютер и повторите проверку.

ВАЖНО: Устанавливайте именно SDK, а не Runtime!
SDK включает в себя всё необходимое для сборки и запуска приложения.


2.3. ЗАПУСК ПРИЛОЖЕНИЯ
───────────────────────────────────────────────────────────────────────────────

Файл для запуска: ЗАПУСК.bat

Это единственный файл, который вам нужен для запуска приложения!

Что делает ЗАПУСК.bat:
1. Проверяет наличие .NET SDK на компьютере
2. Отображает версию установленного .NET SDK
3. Ищет готовый скомпилированный .exe файл
4. Если файл найден - запускает его сразу (1-5 секунд)
5. Если файл не найден - автоматически собирает проект (1-3 минуты)
6. Запускает приложение
7. Показывает понятные сообщения об ошибках (если они возникли)

Как запустить:
   Способ 1 (Рекомендуется):
   • Найдите файл ЗАПУСК.bat в корневой папке проекта
   • Двойной клик по файлу
   • Дождитесь открытия окна входа

   Способ 2 (Через контекстное меню):
   • Щёлкните правой кнопкой мыши на файле ЗАПУСК.bat
   • Выберите "Открыть"
   • Дождитесь открытия окна входа

   Способ 3 (Через командную строку):
   • Откройте командную строку в папке проекта
   • Введите: ЗАПУСК.bat
   • Нажмите Enter

Время запуска:
• Первый раз: 1-3 минуты (автоматическая сборка проекта)
• Последующие разы: 1-5 секунд (запуск готового файла)

Что происходит при первом запуске:
1. Проверка .NET SDK (5 секунд)
2. Восстановление пакетов NuGet (30-60 секунд)
3. Компиляция проекта (30-90 секунд)
4. Создание исполняемого файла (10 секунд)
5. Запуск приложения (5 секунд)
6. Автоматическое создание базы данных (2 секунды)

2.5. РЕШЕНИЕ ПРОБЛЕМ ПРИ УСТАНОВКЕ
───────────────────────────────────────────────────────────────────────────────

Проблема 1: Приложение не запускается

Возможные причины:
• Не установлен .NET SDK
• Установлена неправильная версия .NET
• Повреждены файлы проекта

Решение:
1. Проверьте установку .NET SDK:
   • Откройте командную строку
   • Введите: dotnet --version
   • Должна отобразиться версия 6.0 или выше
2. Если .NET SDK не установлен:
   • Скачайте с https://dotnet.microsoft.com/download/dotnet/6.0
   • Установите .NET 6.0 SDK
   • Перезагрузите компьютер
3. Запустите ЗАПУСК.bat заново

Проблема 2: Ошибка "dotnet не является внутренней или внешней командой"

Причина: .NET SDK не установлен или не добавлен в PATH

Решение:
1. Установите .NET 6.0 SDK
2. Перезагрузите компьютер
3. Проверьте установку: dotnet --version

Проблема 3: Ошибка компиляции проекта

Возможные причины:
• Нет подключения к интернету (при первом запуске)
• Повреждены файлы проекта
• Несовместимая версия .NET SDK

Решение:
1. Проверьте подключение к интернету
2. Запустите ЗАПУСК.bat заново
3. Если не помогло - откройте проект в Visual Studio:
   • Откройте файл DatabaseApp.csproj
   • Нажмите F6 (Build Solution)
   • Посмотрите ошибки в окне Error List

Проблема 4: Окно консоли закрывается сразу после запуска

Причина: Произошла ошибка при запуске

Решение:
1. Откройте командную строку в папке проекта
2. Запустите: ЗАПУСК.bat
3. Прочитайте сообщение об ошибке
4. Следуйте инструкциям в сообщении

Проблема 5: База данных не создаётся

Возможные причины:
• Нет прав доступа к папке
• Недостаточно места на диске
• Антивирус блокирует создание файла

Решение:
1. Запустите ЗАПУСК.bat от имени администратора:
   • Щёлкните правой кнопкой на ЗАПУСК.bat
   • Выберите "Запуск от имени администратора"
2. Проверьте свободное место на диске (нужно минимум 100 МБ)
3. Добавьте папку проекта в исключения антивируса

3.1. ВХОД В СИСТЕМУ

Если вход не удаётся:
1. Проверьте раскладку клавиатуры (должна быть английская)
2. Проверьте, не нажат ли Caps Lock
3. Убедитесь, что вводите логин маленькими буквами
4. Попробуйте скопировать логин и пароль из этого документа

3.6. СОЗДАНИЕ БУХГАЛТЕРСКИХ ПРОВОДОК
───────────────────────────────────────────────────────────────────────────────

Проводки - это записи в бухгалтерском учёте по принципу двойной записи.

Принцип двойной записи:
Каждая хозяйственная операция отражается одновременно по дебету одного счёта
и кредиту другого счёта на одинаковую сумму.

Просмотр журнала проводок:
1. В главном меню выберите "Проводки"
2. Откроется окно со списком всех проводок
3. В таблице отображаются: Дата, Дебет, Кредит, Сумма, Описание

Создание новой проводки:
1. Нажмите кнопку "Добавить"
2. Заполните обязательные поля:
   • Дата проводки (выберите из календаря)
   • Счёт Дебет (выберите из списка счетов)
   • Счёт Кредит (выберите из списка счетов)
   • Сумма (в рублях)
3. Заполните дополнительные поля (необязательно):
   • Документ (выберите из списка документов)
   • Описание (содержание операции)
4. Нажмите кнопку "Сохранить"

Примеры типичных проводок:

1. Получение денег из банка в кассу:
   Дт 50 (Касса) - Кт 51 (Расчётный счёт) = 10000 руб

2. Покупка товара у поставщика:
   Дт 41 (Товары) - Кт 60 (Расчёты с поставщиками) = 5000 руб

3. Оплата поставщику:
   Дт 60 (Расчёты с поставщиками) - Кт 51 (Расчётный счёт) = 5000 руб

4. Продажа товара покупателю:
   Дт 62 (Расчёты с покупателями) - Кт 90 (Продажи) = 8000 руб

5. Списание себестоимости проданного товара:
   Дт 90 (Продажи) - Кт 41 (Товары) = 5000 руб

6. Получение оплаты от покупателя:
   Дт 51 (Расчётный счёт) - Кт 62 (Расчёты с покупателями) = 8000 руб

Важно:
• Счёт Дебет и счёт Кредит должны быть разными
• Сумма должна быть больше 0
• Дата проводки должна быть в пределах отчётного периода

4.1. ЧТО ТАКОЕ ОБОРОТНО-САЛЬДОВАЯ ВЕДОМОСТЬ
───────────────────────────────────────────────────────────────────────────────

Оборотно-сальдовая ведомость (ОСВ) - это основной бухгалтерский отчёт, который
показывает состояние счетов бухгалтерского учёта за определённый период.

Что показывает ОСВ:
• Сальдо начальное - остатки на начало периода (на какую дату начинаем)
• Обороты - движение средств за период (что произошло за период)
• Сальдо конечное - остатки на конец периода (на какую дату заканчиваем)

Для чего нужна ОСВ:
✓ Контроль правильности бухгалтерского учёта
✓ Проверка соблюдения принципа двойной записи
✓ Анализ финансового состояния организации
✓ Подготовка бухгалтерской отчётности
✓ Поиск ошибок в проводках

Принцип работы:
Сальдо конечное = Сальдо начальное + Оборот Дт - Оборот Кт

• Система анализирует все проводки за выбранный период

4.4. СТРУКТУРА ОТЧЁТА
───────────────────────────────────────────────────────────────────────────────

Отчёт представлен в виде таблицы с колонками:

Колонка 1: Счёт
   • Номер счёта бухгалтерского учёта
   • Примеры: 01, 10, 50, 51, 60, 62

Колонка 2: Наименование счёта
   • Название счёта
   • Примеры: Касса, Расчётный счёт, Товары

Колонка 3: Сальдо начальное Дт
   • Остаток по дебету на начало периода
   • Для активных счетов - это остаток имущества

Колонка 4: Сальдо начальное Кт
   • Остаток по кредиту на начало периода
   • Для пассивных счетов - это остаток обязательств

Колонка 5: Оборот Дт
   • Сумма всех дебетовых операций за период
   • Показывает, сколько пришло на счёт

Колонка 6: Оборот Кт
   • Сумма всех кредитовых операций за период
   • Показывает, сколько ушло со счёта

Колонка 7: Сальдо конечное Дт
   • Остаток по дебету на конец периода
   • Рассчитывается автоматически

Колонка 8: Сальдо конечное Кт
   • Остаток по кредиту на конец периода
   • Рассчитывается автоматически

Пример строки отчёта:

┌──────┬──────────────┬──────────────┬──────────┬──────────────┐
│ Счёт │ Наименование │ Сальдо нач.  │ Обороты  │ Сальдо кон.  │
│      │              │  Дт  │  Кт   │ Дт │ Кт  │  Дт  │  Кт   │
├──────┼──────────────┼──────┼───────┼────┼─────┼──────┼───────┤
│  50  │ Касса        │ 1000 │   0   │500 │ 300 │ 1200 │   0   │
└──────┴──────────────┴──────┴───────┴────┴─────┴──────┴───────┘

Как читать эту строку:
• На начало периода в кассе было 1000 рублей
• За период в кассу поступило 500 рублей (оборот Дт)
• За период из кассы выдано 300 рублей (оборот Кт)
• На конец периода в кассе осталось 1200 рублей

Проверка: 1000 + 500 - 300 = 1200 ✓


4.5. ПРОВЕРКА ПРАВИЛЬНОСТИ
───────────────────────────────────────────────────────────────────────────────

Главное правило бухгалтерского учёта:
    ИТОГО ОБОРОТОВ ДТ = ИТОГО ОБОРОТОВ КТ

Где посмотреть итоги:
В нижней части окна отчёта отображаются:
• Итого оборотов Дт: 50 000.00
• Итого оборотов Кт: 50 000.00

Если итоги равны:
✅ ПРАВИЛЬНО - все проводки составлены верно
✅ Принцип двойной записи соблюдён
✅ Можно использовать отчёт для анализа

Если итоги НЕ равны:
❌ ОШИБКА - есть неправильные проводки
❌ Нужно найти и исправить ошибку
❌ Отчёт нельзя использовать

Как найти ошибку:
1. Откройте раздел "Проводки"
2. Просмотрите все проводки за период
3. Проверьте суммы в каждой проводке
4. Найдите проводку с неправильной суммой
5. Исправьте или удалите её
6. Пересформируйте отчёт

Дополнительные проверки:

Проверка 1: Активные счета
• Должны иметь сальдо только по дебету
• Если есть сальдо по кредиту - ошибка

Проверка 2: Пассивные счета
• Должны иметь сальдо только по кредиту
• Если есть сальдо по дебету - ошибка

Проверка 3: Логичность остатков
• Касса не может быть отрицательной
• Расчётный счёт обычно положительный
• Задолженность поставщикам - по кредиту счёта 60

Результат: Финансовый результат деятельности

Пример 5: Анализ дебиторской задолженности

5.1. ОСНОВНЫЕ ПОНЯТИЯ (ДЕБЕТ И КРЕДИТ)
───────────────────────────────────────────────────────────────────────────────

Что такое ДЕБЕТ и КРЕДИТ?

Дебет (Дт) - левая сторона счёта
Кредит (Кт) - правая сторона счёта

Это НЕ "приход" и "расход"! Это просто две стороны счёта.

Принцип двойной записи:
Каждая хозяйственная операция отражается ОДНОВРЕМЕННО:
• По ДЕБЕТУ одного счёта
• По КРЕДИТУ другого счёта
• На ОДИНАКОВУЮ сумму

Пример:
Получили деньги из банка в кассу - 10 000 рублей

Проводка: Дт 50 (Касса) - Кт 51 (Расчётный счёт) = 10 000

Что произошло:
• В кассе стало БОЛЬШЕ на 10 000 (увеличение по дебету активного счёта)
• На расчётном счёте стало МЕНЬШЕ на 10 000 (уменьшение по кредиту активного счёта)

Золотое правило:
    СУММА ВСЕХ ДЕБЕТОВ = СУММА ВСЕХ КРЕДИТОВ

Если это правило нарушено - в учёте ошибка!


5.2. ТИПЫ СЧЕТОВ
───────────────────────────────────────────────────────────────────────────────

В бухгалтерском учёте существует 3 типа счетов:

1. АКТИВНЫЕ СЧЕТА (учёт имущества)

Характеристики:
• Учитывают имущество организации
• Сальдо только ДЕБЕТОВОЕ (или нулевое)
• Увеличение - по ДЕБЕТУ
• Уменьшение - по КРЕДИТУ

Формула:
Сальдо конечное Дт = Сальдо начальное Дт + Оборот Дт - Оборот Кт

Примеры активных счетов:
01 - Основные средства (здания, оборудование)
10 - Материалы (сырьё, материалы)
41 - Товары (товары на складе)
50 - Касса (наличные деньги)
51 - Расчётные счета (деньги в банке)

Пример работы активного счёта 50 (Касса):

Начальное сальдо Дт: 5 000 руб (было в кассе)
Оборот Дт: +3 000 руб (поступило в кассу)
Оборот Кт: -2 000 руб (выдано из кассы)
Конечное сальдо Дт: 6 000 руб (осталось в кассе)

Проверка: 5 000 + 3 000 - 2 000 = 6 000 ✓

2. ПАССИВНЫЕ СЧЕТА (учёт источников)

Характеристики:
• Учитывают источники формирования имущества
• Сальдо только КРЕДИТОВОЕ (или нулевое)
• Увеличение - по КРЕДИТУ
• Уменьшение - по ДЕБЕТУ

Формула:
Сальдо конечное Кт = Сальдо начальное Кт + Оборот Кт - Оборот Дт

Примеры пассивных счетов:
60 - Расчёты с поставщиками (долг поставщикам)
66 - Краткосрочные кредиты (кредиты до 1 года)
67 - Долгосрочные кредиты (кредиты более 1 года)
68 - Расчёты по налогам (долг по налогам)
70 - Расчёты с персоналом (долг по зарплате)
80 - Уставный капитал (вклад учредителей)

Пример работы пассивного счёта 60 (Расчёты с поставщиками):

Начальное сальдо Кт: 10 000 руб (должны поставщикам)
Оборот Кт: +5 000 руб (купили ещё товар в долг)
Оборот Дт: -3 000 руб (оплатили поставщику)
Конечное сальдо Кт: 12 000 руб (должны поставщикам)

Проверка: 10 000 + 5 000 - 3 000 = 12 000 ✓

3. АКТИВНО-ПАССИВНЫЕ СЧЕТА (смешанные)

Характеристики:
• Могут иметь как дебетовое, так и кредитовое сальдо
• Или одновременно оба сальдо (развёрнутое сальдо)
• Увеличение - по дебету или кредиту (зависит от операции)

Примеры активно-пассивных счетов:
62 - Расчёты с покупателями
   Дт - покупатели должны нам
   Кт - мы должны покупателям (аванс)

71 - Расчёты с подотчётными лицами
   Дт - работник должен вернуть деньги
   Кт - работнику нужно выдать деньги

76 - Расчёты с разными дебиторами и кредиторами
   Дт - нам должны
   Кт - мы должны

84 - Нераспределённая прибыль (непокрытый убыток)
   Дт - убыток
   Кт - прибыль

90 - Продажи
   Дт - расходы от продаж
   Кт - доходы от продаж

99 - Прибыли и убытки
   Дт - убытки
   Кт - прибыли

5.5. ФОРМУЛЫ И ПРАВИЛА
───────────────────────────────────────────────────────────────────────────────

ОСНОВНЫЕ ФОРМУЛЫ

1. Расчёт конечного сальдо для АКТИВНОГО счёта:
   
   Сальдо конечное Дт = Сальдо начальное Дт + Оборот Дт - Оборот Кт

   Пример для счёта 50 (Касса):
   Было: 10 000 руб
   Поступило: 5 000 руб
   Выдано: 3 000 руб
   Осталось: 10 000 + 5 000 - 3 000 = 12 000 руб

2. Расчёт конечного сальдо для ПАССИВНОГО счёта:
   
   Сальдо конечное Кт = Сальдо начальное Кт + Оборот Кт - Оборот Дт

   Пример для счёта 60 (Расчёты с поставщиками):
   Должны были: 20 000 руб
   Купили ещё: 10 000 руб
   Оплатили: 8 000 руб
   Должны: 20 000 + 10 000 - 8 000 = 22 000 руб

3. Расчёт финансового результата от продаж:
   
   Финансовый результат = Выручка - Себестоимость - Расходы

   Пример:
   Выручка: 100 000 руб (Кт 90)
   Себестоимость: 60 000 руб (Дт 90)
   Расходы: 10 000 руб (Дт 90)
   Прибыль: 100 000 - 60 000 - 10 000 = 30 000 руб

4. Расчёт НДС:
   
   НДС 20% = Сумма × 20 / 120
   НДС 10% = Сумма × 10 / 110

   Пример:
   Цена с НДС: 120 000 руб
   НДС 20%: 120 000 × 20 / 120 = 20 000 руб
   Цена без НДС: 120 000 - 20 000 = 100 000 руб

ОСНОВНЫЕ ПРАВИЛА

Правило 1: БАЛАНС
   Актив = Пассив
   Имущество = Источники формирования имущества

Правило 2: ДВОЙНАЯ ЗАПИСЬ
   Каждая операция отражается дважды:
   - По дебету одного счёта
   - По кредиту другого счёта
   - На одинаковую сумму

Правило 3: РАВЕНСТВО ОБОРОТОВ
   Сумма оборотов по дебету ВСЕХ счетов = Сумма оборотов по кредиту ВСЕХ счетов

Правило 4: АКТИВНЫЕ СЧЕТА
   - Сальдо только дебетовое
   - Увеличение по дебету
   - Уменьшение по кредиту

Правило 5: ПАССИВНЫЕ СЧЕТА
   - Сальдо только кредитовое
   - Увеличение по кредиту
   - Уменьшение по дебету

Правило 6: АКТИВНО-ПАССИВНЫЕ СЧЕТА
   - Могут иметь дебетовое или кредитовое сальдо
   - Или оба сальдо одновременно

Правило 7: ХРОНОЛОГИЯ
   - Проводки делаются в хронологическом порядке
   - Дата проводки не может быть в будущем
   - Нельзя делать проводки за закрытый период


5.6. КОНТРОЛЬНЫЕ СООТНОШЕНИЯ
───────────────────────────────────────────────────────────────────────────────

Контрольные соотношения - это правила проверки правильности учёта.

ОБЯЗАТЕЛЬНЫЕ ПРОВЕРКИ

1. Равенство оборотов
   ✓ Итого оборотов Дт = Итого оборотов Кт
   
   Если не равны - ОШИБКА в проводках!

2. Равенство начальных сальдо
   ✓ Сумма начальных сальдо Дт = Сумма начальных сальдо Кт
   
   Это проверка баланса на начало периода

3. Равенство конечных сальдо
   ✓ Сумма конечных сальдо Дт = Сумма конечных сальдо Кт
   
   Это проверка баланса на конец периода

4. Проверка активных счетов
   ✓ Сальдо конечное Кт = 0 (или пусто)
   
   У активных счетов не может быть кредитового сальдо!

5. Проверка пассивных счетов
   ✓ Сальдо конечное Дт = 0 (или пусто)
   
   У пассивных счетов не может быть дебетового сальдо!

ЛОГИЧЕСКИЕ ПРОВЕРКИ

6. Касса не может быть отрицательной
   ✓ Счёт 50: Сальдо Дт ≥ 0
   
   Нельзя выдать из кассы больше, чем в ней есть!

7. Расчётный счёт обычно положительный
   ✓ Счёт 51: Сальдо Дт ≥ 0
   
   Овердрафт возможен, но это исключение

8. Уставный капитал не изменяется
   ✓ Счёт 80: Сальдо Кт = const
   
   Изменение уставного капитала - редкая операция

9. Товары не могут быть отрицательными
   ✓ Счёт 41: Сальдо Дт ≥ 0
   
   Нельзя продать больше товара, чем есть на складе!

10. Проверка финансового результата
    ✓ Счёт 90: Оборот Кт (выручка) > Оборот Дт (расходы)
    
    Если расходы больше выручки - это убыток

ПРОВЕРКИ ВЗАИМОСВЯЗЕЙ

11. Продажа товара
    ✓ Если есть Дт 62 - Кт 90 (продажа)
    ✓ Должна быть Дт 90 - Кт 41 (списание себестоимости)
    
    Продали товар - нужно списать его себестоимость!

12. Покупка и оплата
    ✓ Если есть Дт 41 - Кт 60 (покупка товара)
    ✓ Должна быть Дт 60 - Кт 51 (оплата поставщику)
    
    Купили товар - нужно его оплатить!

13. Начисление и выплата зарплаты
    ✓ Если есть Дт 20 - Кт 70 (начисление)
    ✓ Должна быть Дт 70 - Кт 50 (выплата)
    
    Начислили зарплату - нужно её выплатить!

ПРОВЕРКИ ПЕРИОДОВ

14. Соответствие периодов
    ✓ Дата проводки должна быть в пределах отчётного периода
    
    Нельзя делать проводки за пределами периода!

15. Последовательность операций
    ✓ Сначала покупка, потом продажа
    ✓ Сначала начисление, потом выплата
    
    Логическая последовательность операций!

6.4. ПРАКТИЧЕСКИЕ ЗАДАНИЯ
───────────────────────────────────────────────────────────────────────────────

ЗАДАНИЕ 1: Операции с кассой

Условие:
На начало дня в кассе было 5 000 руб. В течение дня:
1. Получено из банка 50 000 руб
2. Выдана зарплата 40 000 руб
3. Получена выручка 15 000 руб
4. Оплачено поставщику 8 000 руб

Задача: Создать проводки и определить остаток в кассе на конец дня.

Решение:
Проводка 1: Дт 50 - Кт 51 = 50 000 (получено из банка)
Проводка 2: Дт 70 - Кт 50 = 40 000 (выдана зарплата)
Проводка 3: Дт 50 - Кт 62 = 15 000 (получена выручка)
Проводка 4: Дт 60 - Кт 50 = 8 000 (оплачено поставщику)

Расчёт остатка:
Начальное сальдо: 5 000
Оборот Дт: 50 000 + 15 000 = 65 000
Оборот Кт: 40 000 + 8 000 = 48 000
Конечное сальдо: 5 000 + 65 000 - 48 000 = 22 000 руб

Ответ: В кассе на конец дня 22 000 руб

ЗАДАНИЕ 2: Покупка и продажа товара

Условие:
1. Куплен товар у поставщика на сумму 60 000 руб (в долг)
2. Оплачено поставщику 60 000 руб с расчётного счёта
3. Продан товар покупателю за 100 000 руб (в долг)
4. Получена оплата от покупателя 100 000 руб на расчётный счёт

Задача: Создать все необходимые проводки.

Решение:
Проводка 1: Дт 41 - Кт 60 = 60 000 (покупка товара)
Проводка 2: Дт 60 - Кт 51 = 60 000 (оплата поставщику)
Проводка 3: Дт 62 - Кт 90 = 100 000 (продажа товара)
Проводка 4: Дт 90 - Кт 41 = 60 000 (списание себестоимости)
Проводка 5: Дт 51 - Кт 62 = 100 000 (получена оплата)

Финансовый результат:
Выручка: 100 000 (Кт 90)
Себестоимость: 60 000 (Дт 90)
Прибыль: 100 000 - 60 000 = 40 000 руб

ЗАДАНИЕ 3: Расчёты с персоналом

Условие:
1. Начислена зарплата работникам 200 000 руб
2. Удержан НДФЛ 13% = 26 000 руб
3. Выдана зарплата на руки

Задача: Создать проводки и рассчитать сумму к выдаче.

Решение:
Проводка 1: Дт 20 - Кт 70 = 200 000 (начисление зарплаты)
Проводка 2: Дт 70 - Кт 68 = 26 000 (удержание НДФЛ)
Проводка 3: Дт 70 - Кт 50 = 174 000 (выдача зарплаты)

Расчёт:
Начислено: 200 000
Удержано НДФЛ: 26 000
К выдаче: 200 000 - 26 000 = 174 000 руб

ЗАДАНИЕ 4: Формирование финансового результата

Условие:
За месяц:
- Выручка от продаж: 500 000 руб
- Себестоимость проданных товаров: 300 000 руб
- Коммерческие расходы: 50 000 руб
- Управленческие расходы: 80 000 руб

Задача: Определить финансовый результат.

Решение:
Валовая прибыль = Выручка - Себестоимость
Валовая прибыль = 500 000 - 300 000 = 200 000 руб

Прибыль от продаж = Валовая прибыль - Коммерческие расходы - Управленческие расходы
Прибыль от продаж = 200 000 - 50 000 - 80 000 = 70 000 руб

Проводка: Дт 90 - Кт 99 = 70 000 (прибыль от продаж)

Ответ: Прибыль от продаж 70 000 руб

ОШИБКА 2: Не могу войти в систему

Симптомы:
• Ввожу логин и пароль, но вход не происходит
• Появляется сообщение "Неверный логин или пароль"

Причины:
• Неправильный логин или пароль
• Включен Caps Lock
• Русская раскладка клавиатуры

Решение:
1. Проверьте раскладку (должна быть английская)
2. Проверьте Caps Lock (должен быть выключен)
3. Используйте: admin / admin123 (маленькими буквами)
4. Скопируйте логин и пароль из этого документа

ОШИБКА 3: Итого Дт ≠ Итого Кт в отчёте

Симптомы:
• В оборотно-сальдовой ведомости итоги не равны
• Итого оборотов Дт: 100 000
• Итого оборотов Кт: 95 000

Причины:
• Ошибка в проводках
• Неправильно указана сумма
• Удалена часть проводки

Решение:
1. Откройте раздел "Проводки"
2. Просмотрите все проводки за период
3. Найдите проводку с неправильной суммой
4. Исправьте или удалите её
5. Пересформируйте отчёт

ОШИБКА 4: Отрицательный остаток в кассе

Симптомы:
• В отчёте счёт 50 (Касса) имеет кредитовое сальдо
• Или сальдо Дт отрицательное

Причины:
• Выдано из кассы больше, чем было
• Забыли отразить поступление в кассу
• Неправильная последовательность проводок

Решение:
1. Проверьте все проводки по кассе
2. Убедитесь, что сначала деньги поступили, потом выданы
3. Добавьте недостающие проводки поступления
4. Пересформируйте отчёт

ОШИБКА 5: Не могу экспортировать отчёт

Симптомы:
• Нажимаю "Экспорт в CSV", но ничего не происходит
• Появляется ошибка при сохранении

Причины:
• Нет прав на запись в выбранную папку
• Файл с таким именем уже открыт в Excel
• Недостаточно места на диске

Решение:
1. Выберите другую папку для сохранения (например, Рабочий стол)
2. Закройте Excel, если он открыт
3. Используйте другое имя файла
4. Проверьте свободное место на диске

ОШИБКА 6: Забыл пароль

Симптомы:
• Не помню пароль от учётной записи

Решение:

• admin / admin123

3. База данных создастся с начальными паролями

ОШИБКА 7: Проводка не сохраняется

• Заполняю все поля, нажимаю "Сохранить"
• Проводка не появляется в списке

Причины:
• Не заполнены обязательные поля

2. Убедитесь, что Дебет ≠ Кредит

7.3. АНАЛИЗ ФИНАНСОВОГО СОСТОЯНИЯ
───────────────────────────────────────────────────────────────────────────────

ПОКАЗАТЕЛИ ЛИКВИДНОСТИ

1. Коэффициент текущей ликвидности
   Формула: Оборотные активы / Краткосрочные обязательства
   Норма: ≥ 2
   Что показывает: Способность погасить краткосрочные обязательства

   Пример расчёта:
   Оборотные активы (счета 10, 41, 50, 51, 62): 500 000 руб
   Краткосрочные обязательства (счета 60, 66, 70): 200 000 руб
   Коэффициент: 500 000 / 200 000 = 2,5 (ХОРОШО)

2. Коэффициент быстрой ликвидности
   Формула: (Денежные средства + Дебиторская задолженность) / Краткосрочные обязательства
   Норма: ≥ 1
   Что показывает: Способность быстро погасить долги

   Пример расчёта:
   Денежные средства (счета 50, 51): 100 000 руб
   Дебиторская задолженность (счёт 62): 150 000 руб
   Краткосрочные обязательства: 200 000 руб
   Коэффициент: (100 000 + 150 000) / 200 000 = 1,25 (ХОРОШО)

3. Коэффициент абсолютной ликвидности
   Формула: Денежные средства / Краткосрочные обязательства
   Норма: ≥ 0,2
   Что показывает: Способность немедленно погасить долги

   Пример расчёта:
   Денежные средства (счета 50, 51): 100 000 руб
   Краткосрочные обязательства: 200 000 руб
   Коэффициент: 100 000 / 200 000 = 0,5 (ОТЛИЧНО)

ПОКАЗАТЕЛИ РЕНТАБЕЛЬНОСТИ

4. Рентабельность продаж
   Формула: (Прибыль от продаж / Выручка) × 100%
   Норма: > 0
   Что показывает: Сколько прибыли с каждого рубля выручки

   Пример расчёта:
   Выручка (Кт 90): 1 000 000 руб
   Себестоимость (Дт 90): 600 000 руб
   Расходы (Дт 90): 200 000 руб
   Прибыль: 1 000 000 - 600 000 - 200 000 = 200 000 руб
   Рентабельность: (200 000 / 1 000 000) × 100% = 20%

5. Рентабельность активов (ROA)
   Формула: (Чистая прибыль / Активы) × 100%
   Норма: > 0
   Что показывает: Эффективность использования активов

   Пример расчёта:
   Чистая прибыль (Кт 99): 150 000 руб
   Активы (сумма всех активных счетов): 1 500 000 руб
   ROA: (150 000 / 1 500 000) × 100% = 10%

ПОКАЗАТЕЛИ ФИНАНСОВОЙ УСТОЙЧИВОСТИ

6. Коэффициент автономии
   Формула: Собственный капитал / Активы
   Норма: ≥ 0,5
   Что показывает: Долю собственных средств в активах

   Пример расчёта:
   Собственный капитал (счета 80, 82, 83, 84): 800 000 руб
   Активы: 1 500 000 руб
   Коэффициент: 800 000 / 1 500 000 = 0,53 (ХОРОШО)

7. Коэффициент финансовой зависимости
   Формула: Заёмный капитал / Активы
   Норма: ≤ 0,5
   Что показывает: Долю заёмных средств в активах

   Пример расчёта:
   Заёмный капитал (счета 60, 66, 67, 70): 700 000 руб
   Активы: 1 500 000 руб
   Коэффициент: 700 000 / 1 500 000 = 0,47 (ХОРОШО)

ПОКАЗАТЕЛИ ДЕЛОВОЙ АКТИВНОСТИ

8. Оборачиваемость товаров
   Формула: Себестоимость продаж / Средний остаток товаров
   Что показывает: Сколько раз товар оборачивается за период

   Пример расчёта:
   Себестоимость продаж (Дт 90 - Кт 41): 600 000 руб
   Средний остаток товаров (счёт 41): 100 000 руб
   Оборачиваемость: 600 000 / 100 000 = 6 раз

9. Период оборота товаров
   Формула: 365 дней / Оборачиваемость товаров
   Что показывает: Сколько дней товар лежит на складе

   Пример расчёта:
   Оборачиваемость: 6 раз
   Период: 365 / 6 = 61 день

10. Оборачиваемость дебиторской задолженности
    Формула: Выручка / Средняя дебиторская задолженность
    Что показывает: Как быстро покупатели платят

    Пример расчёта:
    Выручка (Кт 90): 1 000 000 руб
    Средняя дебиторская задолженность (счёт 62): 150 000 руб
    Оборачиваемость: 1 000 000 / 150 000 = 6,67 раз
    Период: 365 / 6,67 = 55 дней


7.4. РЕШЕНИЕ ПРОБЛЕМ
───────────────────────────────────────────────────────────────────────────────

ПРОБЛЕМА: Приложение не запускается

Симптом: Двойной клик по ЗАПУСК.bat не открывает приложение

Диагностика:
1. Откройте командную строку (Win+R, cmd, Enter)
2. Введите: dotnet --version
3. Нажмите Enter

Если команда не найдена:
→ Решение: Установите .NET 6.0 SDK с сайта Microsoft
→ Ссылка: https://dotnet.microsoft.com/download/dotnet/6.0
→ После установки перезагрузите компьютер

Если версия отображается:
→ Решение: Запустите ЗАПУСК.bat от имени администратора
→ Щёлкните правой кнопкой → "Запуск от имени администратора"

ПРОБЛЕМА: Ошибка при компиляции

Симптом: При запуске ЗАПУСК.bat появляется ошибка компиляции

Возможные причины:
• Нет подключения к интернету (при первом запуске)
• Повреждены файлы проекта
• Несовместимая версия .NET SDK

Решение 1: Проверьте интернет-соединение
→ Откройте браузер и проверьте доступ к интернету
→ При первом запуске нужен интернет для загрузки пакетов NuGet

Решение 2: Очистите кэш NuGet
→ Откройте командную строку
→ Введите: dotnet nuget locals all --clear
→ Запустите ЗАПУСК.bat заново

Решение 3: Восстановите пакеты вручную
→ Откройте командную строку в папке DatabaseApp\DatabaseApp
→ Введите: dotnet restore
→ Дождитесь завершения
→ Введите: dotnet build
→ Запустите ЗАПУСК.bat

ПРОБЛЕМА: База данных не создаётся

Симптом: Приложение запускается, но нет данных в справочниках

Диагностика:
→ Проверьте наличие файла accounting.db в папке:
   DatabaseApp\DatabaseApp\bin\Release\net6.0-windows\

Если файла нет:
Решение 1: Проверьте права доступа
→ Запустите приложение от имени администратора
→ База данных должна создаться автоматически

Решение 2: Проверьте свободное место на диске
→ Нужно минимум 100 МБ свободного места
→ Очистите диск, если места недостаточно

Решение 3: Проверьте антивирус
→ Добавьте папку проекта в исключения антивируса
→ Антивирус может блокировать создание файла БД

ПРОБЛЕМА: Не могу войти в систему

Симптом: Ввожу логин и пароль, но вход не происходит

Решение:
→ Проверьте раскладку клавиатуры (должна быть английская)
→ Проверьте Caps Lock (должен быть выключен)
→ Используйте точные учётные данные:
   Логин: admin (маленькими буквами)
   Пароль: admin123 (маленькими буквами)

Если не помогает:
→ Удалите файл accounting.db
→ Запустите приложение заново
→ База данных создастся с начальными паролями

ПРОБЛЕМА: Итого Дт ≠ Итого Кт

Симптом: В оборотно-сальдовой ведомости итоги не равны

Причина: Ошибка в проводках

Решение:
1. Откройте раздел "Проводки"
2. Отсортируйте по дате
3. Проверьте каждую проводку:
   • Правильно ли указаны счета Дт и Кт
   • Правильно ли указана сумма
   • Не удалена ли часть проводки
4. Найдите и исправьте ошибочную проводку
5. Пересформируйте отчёт

Совет: Проверяйте отчёт после каждой новой проводки!

ПРОБЛЕМА: Отрицательный остаток в кассе

Симптом: Счёт 50 имеет кредитовое сальдо или отрицательное Дт

Причина: Выдано из кассы больше, чем было

Решение:
1. Откройте раздел "Проводки"
2. Найдите все проводки по счёту 50
3. Проверьте последовательность:
   • Сначала должны быть поступления (Дт 50)
   • Потом выдачи (Кт 50)
4. Добавьте недостающие проводки поступления
5. Или исправьте суммы выдач

Правило: В кассе не может быть отрицательного остатка!

ПРОБЛЕМА: Не открывается экспортированный CSV файл

Симптом: Файл CSV не открывается в Excel или отображается неправильно

Решение 1: Открыть через Excel
→ Откройте Excel
→ Файл → Открыть
→ Выберите "Все файлы (*.*)"
→ Найдите и откройте CSV файл
→ Выберите разделитель "Точка с запятой"

Решение 2: Изменить региональные настройки
→ Панель управления → Региональные стандарты
→ Дополнительные параметры
→ Разделитель элементов списка: ;
→ Нажмите ОК

ПРОБЛЕМА: Медленная работа приложения

Симптом: Приложение долго открывается или тормозит

Возможные причины:
• Большой объём данных в базе
• Недостаточно оперативной памяти
• Антивирус сканирует файлы

Решение 1: Очистите старые данные
→ Удалите ненужные проводки
→ Удалите старые документы
→ Оставьте только актуальные данные

Решение 2: Добавьте в исключения антивируса
→ Добавьте папку проекта в исключения
→ Добавьте файл DatabaseApp.exe в исключения

Решение 3: Увеличьте оперативную память
→ Закройте ненужные программы
→ Перезагрузите компьютер


7.5. ПОЛЕЗНЫЕ СОВЕТЫ
───────────────────────────────────────────────────────────────────────────────

СОВЕТ 1: Регулярно проверяйте отчёт

После каждой новой проводки формируйте оборотно-сальдовую ведомость и проверяйте:
✓ Итого Дт = Итого Кт
✓ Нет отрицательных остатков в кассе

✓ Нет дебетового сальдо на пассивных счетах

Это поможет сразу обнаружить ошибку, а не искать её потом среди сотен проводок!

СОВЕТ 2: Используйте описания в проводках

Всегда заполняйте поле "Описание" при создании проводки:
✓ "Получен товар от ООО Поставщик по накладной №123"
✓ "Оплачено поставщику за товар по счёту №456"
✓ "Выдана зарплата за май 2026"

Это поможет понять смысл проводки через месяц или год!

СОВЕТ 3: Создавайте резервные копии

Регулярно копируйте файл accounting.db в безопасное место:
✓ На флешку

✓ На другой компьютер

1. Сначала покупка товара (Дт 41 - Кт 60)

3. Потом продажа товара (Дт 62 - Кт 90)
4. Потом списание себестоимости (Дт 90 - Кт 41)
5. Потом получение оплаты (Дт 51 - Кт 62)

✓ План счетов с описаниями

СОВЕТ 6: Проверяйте логику операций

Перед созданием проводки подумайте:
• Что увеличивается? → Это пойдёт в Дебет (для активных) или Кредит (для пассивных)
• Что уменьшается? → Это пойдёт в Кредит (для активных) или Дебет (для пассивных)

Пример: Купили товар в долг
• Увеличились товары (актив) → Дт 41
• Увеличился долг поставщику (пассив) → Кт 60

СОВЕТ 10: Помните о времени

using System;
using System.Windows;
using DatabaseApp.Models;

namespace DatabaseApp
{
    public partial class LoginWindow : Window
    {
        public User? CurrentUser { get; private set; }

        public LoginWindow()
        {
            InitializeComponent();
            
            try
            {
                DatabaseHelper.TestConnection();
            }
            catch (Exception ex)
            {
                MessageBox.Show($"Ошибка инициализации базы данных: {ex.Message}", 
                    "Ошибка", MessageBoxButton.OK, MessageBoxImage.Error);
            }
        }

        private void LoginButton_Click(object sender, RoutedEventArgs e)
        {
            string username = UsernameTextBox.Text.Trim();
            string password = PasswordBox.Password;

            if (string.IsNullOrEmpty(username) || string.IsNullOrEmpty(password))
            {
                ErrorTextBlock.Text = "Введите имя пользователя и пароль";
                return;
            }

            try
            {
                string query = $"SELECT * FROM users WHERE username = '{username}' AND password_hash = '{password}'";
                var dataTable = DatabaseHelper.ExecuteQuery(query);

                if (dataTable.Rows.Count > 0)
                {
                    var row = dataTable.Rows[0];
                    CurrentUser = new User
                    {
                        UserId = Convert.ToInt32(row["user_id"]),
                        Username = row["username"].ToString() ?? "",
                        Role = row["role"].ToString() ?? "",
                        FullName = row["full_name"].ToString() ?? ""
                    };

                    string updateQuery = $"UPDATE users SET last_login = datetime('now') WHERE user_id = {CurrentUser.UserId}";
                    DatabaseHelper.ExecuteNonQuery(updateQuery);

                    MainWindow mainWindow = new MainWindow(CurrentUser);
                    mainWindow.Show();
                    this.Close();
                }
                else
                {
                    ErrorTextBlock.Text = "Неверное имя пользователя или пароль";
                }
            }
            catch (Exception ex)
            {
                ErrorTextBlock.Text = $"Ошибка: {ex.Message}";
            }
        }
    }
}

        private List<BalanceSheetRow> GenerateBalanceSheet(DateTime startDate, DateTime endDate)
        {
            var result = new List<BalanceSheetRow>();

            // Получение всех счетов
            string accountsQuery = "SELECT account_number, account_name, account_type FROM accounts ORDER BY account_number";
            DataTable accountsTable = DatabaseHelper.ExecuteQuery(accountsQuery);

            foreach (DataRow accountRow in accountsTable.Rows)
            {
                string accountNumber = accountRow["account_number"].ToString() ?? "";
                string accountName = accountRow["account_name"].ToString() ?? "";
                string accountType = accountRow["account_type"].ToString() ?? "";

                // Начальное сальдо (до начала периода)
                var openingBalance = CalculateBalance(accountNumber, DateTime.MinValue, startDate.AddDays(-1));

                // Обороты за период
                var turnover = CalculateTurnover(accountNumber, startDate, endDate);

                // Конечное сальдо
                decimal closingDebit = 0;
                decimal closingCredit = 0;

                if (accountType == "active")
                {
                    // Активный счёт
                    decimal balance = openingBalance.Debit - openingBalance.Credit + turnover.Debit - turnover.Credit;
                    if (balance > 0)
                        closingDebit = balance;
                    else if (balance < 0)
                        closingCredit = Math.Abs(balance);
                }
                else if (accountType == "passive")
                {
                    // Пассивный счёт
                    decimal balance = openingBalance.Credit - openingBalance.Debit + turnover.Credit - turnover.Debit;
                    if (balance > 0)
                        closingCredit = balance;
                    else if (balance < 0)
                        closingDebit = Math.Abs(balance);
                }
                else // active-passive
                {
                    // Активно-пассивный счёт
                    decimal debitBalance = openingBalance.Debit + turnover.Debit;
                    decimal creditBalance = openingBalance.Credit + turnover.Credit;
                    
                    if (debitBalance > creditBalance)
                        closingDebit = debitBalance - creditBalance;
                    else if (creditBalance > debitBalance)
                        closingCredit = creditBalance - debitBalance;
                }

                // Добавляем строку только если есть обороты или сальдо
                if (openingBalance.Debit != 0 || openingBalance.Credit != 0 || 
                    turnover.Debit != 0 || turnover.Credit != 0 ||
                    closingDebit != 0 || closingCredit != 0)
                {
                    result.Add(new BalanceSheetRow
                    {
                        AccountNumber = accountNumber,
                        AccountName = accountName,
                        OpeningBalanceDebit = openingBalance.Debit,
                        OpeningBalanceCredit = openingBalance.Credit,
                        TurnoverDebit = turnover.Debit,
                        TurnoverCredit = turnover.Credit,
                        ClosingBalanceDebit = closingDebit,
                        ClosingBalanceCredit = closingCredit
                    });
                }
            }

            return result;
        }

        private (decimal Debit, decimal Credit) CalculateBalance(string accountNumber, DateTime startDate, DateTime endDate)
        {
            decimal debit = 0;
            decimal credit = 0;

            try
            {
                // Дебетовые обороты
                string debitQuery = $@"
                    SELECT COALESCE(SUM(amount), 0) as total 
                    FROM transactions 
                    WHERE debit_account = '{accountNumber}' 
                    AND transaction_date >= '{startDate:yyyy-MM-dd}' 
                    AND transaction_date <= '{endDate:yyyy-MM-dd}'";
                
                var debitResult = DatabaseHelper.ExecuteScalar(debitQuery);
                if (debitResult != null && debitResult != DBNull.Value)
                    debit = Convert.ToDecimal(debitResult);

                // Кредитовые обороты
                string creditQuery = $@"
                    SELECT COALESCE(SUM(amount), 0) as total 
                    FROM transactions 
                    WHERE credit_account = '{accountNumber}' 
                    AND transaction_date >= '{startDate:yyyy-MM-dd}' 
                    AND transaction_date <= '{endDate:yyyy-MM-dd}'";
                
                var creditResult = DatabaseHelper.ExecuteScalar(creditQuery);
                if (creditResult != null && creditResult != DBNull.Value)
                    credit = Convert.ToDecimal(creditResult);
            }
            catch
            {
                // Если нет данных, возвращаем нули
            }

            return (debit, credit);
        }

        private (decimal Debit, decimal Credit) CalculateTurnover(string accountNumber, DateTime startDate, DateTime endDate)
        {
            return CalculateBalance(accountNumber, startDate, endDate);
        }

        private void ExportButton_Click(object sender, RoutedEventArgs e)

                    FileName = $"Оборотно-сальдовая_ведомость_{DateTime.Now:yyyy-MM-dd}.csv"

                    MessageBox.Show($"Отчёт успешно экспортирован в файл:\n{saveFileDialog.FileName}", 
                        "Успех", MessageBoxButton.OK, MessageBoxImage.Information);
                }
            }
            catch (Exception ex)
            {
                MessageBox.Show($"Ошибка экспорта: {ex.Message}", "Ошибка", 
                    MessageBoxButton.OK, MessageBoxImage.Error);
            }
        }

        private void ExportToCsv(string filePath)
        {
            var sb = new StringBuilder();
            
            // Заголовок
            sb.AppendLine("Счёт;Наименование счёта;Сальдо начальное Дт;Сальдо начальное Кт;Оборот Дт;Оборот Кт;Сальдо конечное Дт;Сальдо конечное Кт");

            // Данные
            foreach (BalanceSheetRow row in ReportDataGrid.Items)
            {
                sb.AppendLine($"{row.AccountNumber};{row.AccountName};{row.OpeningBalanceDebit:N2};{row.OpeningBalanceCredit:N2};{row.TurnoverDebit:N2};{row.TurnoverCredit:N2};{row.ClosingBalanceDebit:N2};{row.ClosingBalanceCredit:N2}");
            }

            // Итоги
            sb.AppendLine();
            sb.AppendLine($"Итого оборотов:;;;;{TotalDebitTextBlock.Text};{TotalCreditTextBlock.Text};;");

            File.WriteAllText(filePath, sb.ToString(), Encoding.UTF8);
        }

        private void LoadData()
        {
            try
            {
                string query = "SELECT * FROM counterparties ORDER BY name";
                DataTable dataTable = DatabaseHelper.ExecuteQuery(query);

                List<Counterparty> counterparties = new List<Counterparty>();
                foreach (DataRow row in dataTable.Rows)
                {
                    counterparties.Add(new Counterparty
                    {
                        CounterpartyId = Convert.ToInt32(row["counterparty_id"]),
                        Name = row["name"].ToString() ?? "",
                        Inn = row["inn"].ToString() ?? "",
                        Kpp = row["kpp"].ToString() ?? "",
                        Address = row["address"].ToString() ?? "",
                        Phone = row["phone"].ToString() ?? "",
                        Email = row["email"].ToString() ?? "",
                        ContactPerson = row["contact_person"].ToString() ?? "",
                        CounterpartyType = row["counterparty_type"].ToString() ?? ""
                    });
                }

                CounterpartiesDataGrid.ItemsSource = counterparties;
            }
            catch (Exception ex)
            {
                MessageBox.Show($"Ошибка загрузки данных: {ex.Message}", "Ошибка", 
                    MessageBoxButton.OK, MessageBoxImage.Error);
            }
        }

                LoadData();
            }
        }

        private void EditButton_Click(object sender, RoutedEventArgs e)

            if (CounterpartiesDataGrid.SelectedItem is Counterparty selected)

        private void SaveButton_Click(object sender, RoutedEventArgs e)
        {
            if (!DatePicker.SelectedDate.HasValue)
            {
                MessageBox.Show("Выберите дату проводки", "Внимание", 
                    MessageBoxButton.OK, MessageBoxImage.Warning);
                return;
            }

            if (string.IsNullOrWhiteSpace(DebitAccountTextBox.Text))
            {
                MessageBox.Show("Введите счет дебета", "Внимание", 
                    MessageBoxButton.OK, MessageBoxImage.Warning);
                return;
            }

            if (string.IsNullOrWhiteSpace(CreditAccountTextBox.Text))
            {
                MessageBox.Show("Введите счет кредита", "Внимание", 
                    MessageBoxButton.OK, MessageBoxImage.Warning);
                return;
            }

            if (!decimal.TryParse(AmountTextBox.Text, out decimal amount))
            {
                MessageBox.Show("Введите корректную сумму", "Внимание", 
                    MessageBoxButton.OK, MessageBoxImage.Warning);
                return;
            }

            try
            {
                string date = DatePicker.SelectedDate.Value.ToString("yyyy-MM-dd");
                string debitAccount = DebitAccountTextBox.Text.Trim();
                string creditAccount = CreditAccountTextBox.Text.Trim();
                string description = DescriptionTextBox.Text.Trim().Replace("'", "''");

                string query = $@"INSERT INTO transactions 
                    (transaction_date, debit_account, credit_account, amount, description, created_by)
                    VALUES ('{date}', '{debitAccount}', '{creditAccount}', 
                    {amount.ToString().Replace(",", ".")}, '{description}', {currentUser.UserId})";

                DatabaseHelper.ExecuteNonQuery(query);
                MessageBox.Show("Проводка сохранена", "Успех", 
                    MessageBoxButton.OK, MessageBoxImage.Information);
                DialogResult = true;
                Close();
            }
            catch (Exception ex)
            {
                MessageBox.Show($"Ошибка сохранения: {ex.Message}", "Ошибка", 
                    MessageBoxButton.OK, MessageBoxImage.Error);
            }
        }

        private void CancelButton_Click(object sender, RoutedEventArgs e)
        {
            DialogResult = false;
            Close();
        }

8.11. ЧАСТЫЕ ОШИБКИ ПРИ СОЗДАНИИ ПРИЛОЖЕНИЯ
═══════════════════════════════════════════════════════════════════════════════

ОШИБКА 1: "The name 'DatabaseHelper' does not exist in the current context"

ПРИЧИНА: Не создан класс DatabaseHelper или неправильное пространство имён

РЕШЕНИЕ:
1. Проверьте, что файл DatabaseHelper.cs существует
2. Проверьте namespace в файле: должно быть namespace DatabaseApp
3. Пересоберите проект (Build → Rebuild Solution)

───────────────────────────────────────────────────────────────────────────────

ОШИБКА 2: "The type or namespace name 'SQLite' could not be found"

ПРИЧИНА: Не установлен пакет System.Data.SQLite.Core

РЕШЕНИЕ:
1. Tools → NuGet Package Manager → Package Manager Console
2. Введите: Install-Package System.Data.SQLite.Core
3. Дождитесь установки
4. Пересоберите проект

───────────────────────────────────────────────────────────────────────────────

ОШИБКА 3: "InitializeComponent does not exist in the current context"

ПРИЧИНА: Не совпадают имена классов в XAML и C# файлах

РЕШЕНИЕ:
1. Откройте XAML файл
2. Проверьте x:Class="DatabaseApp.LoginWindow"
3. Откройте .cs файл
4. Проверьте namespace DatabaseApp и class LoginWindow
5. Имена должны совпадать!

───────────────────────────────────────────────────────────────────────────────

ОШИБКА 4: "Cannot find resource named 'StartupUri'"

ПРИЧИНА: Неправильно указано стартовое окно в App.xaml

РЕШЕНИЕ:
1. Откройте App.xaml
2. Найдите StartupUri="..."
3. Убедитесь, что указано правильное имя окна
4. Для окна входа: StartupUri="LoginWindow.xaml"

───────────────────────────────────────────────────────────────────────────────

ОШИБКА 5: Приложение запускается, но база данных не создаётся

ПРИЧИНА: Ошибка в коде DatabaseHelper или нет прав на создание файла

РЕШЕНИЕ:
1. Запустите Visual Studio от имени администратора
2. Проверьте код метода InitializeDatabase()
3. Проверьте, что вызывается CreateTables() и InsertInitialData()
4. Проверьте путь к БД: должен быть AppDomain.CurrentDomain.BaseDirectory

───────────────────────────────────────────────────────────────────────────────

ОШИБКА 6: "Object reference not set to an instance of an object"

ПРИЧИНА: Обращение к null объекту

РЕШЕНИЕ:
1. Проверьте, что все элементы в XAML имеют x:Name
2. Проверьте, что InitializeComponent() вызывается в конструкторе
3. Используйте отладчик (F9 - точка останова, F5 - запуск)
4. Проверьте, где именно возникает ошибка

───────────────────────────────────────────────────────────────────────────────

ОШИБКА 7: Кнопки не работают (клик не вызывает метод)

ПРИЧИНА: Не указан обработчик события или неправильное имя метода

РЕШЕНИЕ:
1. В XAML проверьте: Click="LoginButton_Click"
2. В .cs файле проверьте: private void LoginButton_Click(object sender, RoutedEventArgs e)
3. Имена должны совпадать!
4. Метод должен иметь правильную сигнатуру (параметры)
