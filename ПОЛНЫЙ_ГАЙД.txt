1.1  ПРИВЕТСТВИЕ И ОБЗОР ПРОЕКТА
───────────────────────────────────────────────────────────────────────────────

🎓 ДОБРО ПОЖАЛОВАТЬ!

Система бухгалтерского учёта - полнофункциональное WPF приложение для 
демонстрационного экзамена БОМ 26. Приложение ГОТОВО к запуску и демонстрации!

✨ КЛЮЧЕВЫЕ ВОЗМОЖНОСТИ:
  ✓ Полная система бухгалтерского учёта
  ✓ Управление контрагентами и заказчиками
  ✓ Учёт продукции и материалов (8 позиций)
  ✓ Заказы покупателей с позициями
  ✓ Производство продукции с материалами
  ✓ Спецификации продукции
  ✓ Расчёт себестоимости продукции
  ✓ Автоматическое создание БД SQLite
  ✓ Импорт данных из Excel файлов
  ✓ Система авторизации (admin/user)
  ✓ Прокручиваемое меню (колёсиком мыши!)

🛠️ ТЕХНОЛОГИИ:
  • Язык: C# 10
  • Платформа: .NET 6.0
  • Интерфейс: WPF (Windows Presentation Foundation)
  • База данных: SQLite 3 (автономная, без сервера!)
  • ORM: ADO.NET

📊 СТАТИСТИКА ПРОЕКТА:
  • Таблиц в БД: 10 основных + 4 связующих = 14 таблиц
  • Записей в БД: 50+ записей
  • Окон приложения: 10 окон
  • Моделей данных: 10 моделей
  • Строк кода: ~4000+
  • Размер БД: ~150 КБ

1.4  О БАЗЕ ДАННЫХ
───────────────────────────────────────────────────────────────────────────────

🗄️ БАЗА ДАННЫХ SQLite:

   Тип:                       SQLite 3 (встроенная)
   Файл:                      accounting.db
   Расположение:              DatabaseApp\bin\Debug\net6.0-windows\
   Размер:                    ~150 КБ
   Таблиц:                    14 (10 основных + 4 связующих)
   Записей:                   50+
   Связей (FOREIGN KEY):      6
   Индексов:                  Автоматические (PRIMARY KEY)

✨ АВТОМАТИЧЕСКОЕ СОЗДАНИЕ:

   При первом запуске приложения:
   1. Проверяется наличие файла accounting.db
   2. Если файла нет - создаётся новый
   3. Создаются все 14 таблиц
   4. Вставляются начальные данные (50+ записей)
   5. Устанавливаются связи между таблицами
   
   Всё это происходит автоматически!
   Не нужно импортировать SQL скрипты или настраивать подключение!

📊 ТАБЛИЦЫ БАЗЫ ДАННЫХ:

   ОСНОВНЫЕ ТАБЛИЦЫ (10):
   1. users                   - Пользователи системы (2 записи)
   2. customers               - Заказчики (6 организаций)
   3. products                - Продукция и материалы (8 позиций)
   4. customer_orders         - Заказы покупателей (1 заказ)
   5. production              - Производство (1 запись)
   6. specifications          - Спецификации (1 спецификация)
   7. cost_calculations       - Расчёт стоимости (3 расчёта)
   8. counterparties          - Контрагенты (6 организаций) [НЕ ИСПОЛЬЗУЕТСЯ]
   9. documents               - Документы (пусто, готово для заполнения)
   10. transactions           - Транзакции (пусто, готово для заполнения)

   СВЯЗУЮЩИЕ ТАБЛИЦЫ (4):
   11. order_items            - Позиции заказов (3 позиции)
   12. production_materials   - Материалы производства (2 материала)
   13. specification_materials - Материалы спецификаций (2 материала)
   14. accounts               - План счетов [ЕСЛИ ЕСТЬ]

🔗 СВЯЗИ МЕЖДУ ТАБЛИЦАМИ:

   customer_orders.customer_id → customers.customer_id
   order_items.order_id → customer_orders.order_id (CASCADE)
   production_materials.production_id → production.production_id (CASCADE)
   specification_materials.spec_id → specifications.spec_id (CASCADE)
   documents.counterparty_id → counterparties.counterparty_id [ЕСЛИ ЕСТЬ]
   transactions.document_id → documents.document_id (CASCADE) [ЕСЛИ ЕСТЬ]

💡 ВАЖНО: База данных SQLite не требует установки MySQL, PostgreSQL
   или других серверов. Это автономный файл, который работает сразу!

2.2  УСТАНОВКА .NET SDK
───────────────────────────────────────────────────────────────────────────────

📥 СКАЧИВАНИЕ:

   1. Откройте браузер
   2. Перейдите на: https://dotnet.microsoft.com/download/dotnet/6.0
   3. Найдите раздел ".NET SDK 6.0.xxx"
   4. Скачайте установщик для Windows (x64)
   5. Размер: ~150-200 МБ

📦 УСТАНОВКА:

   1. Запустите скачанный файл (dotnet-sdk-6.0.xxx-win-x64.exe)
   2. Нажмите "Install"
   3. Дождитесь завершения установки (2-5 минут)
   4. Нажмите "Close"
   5. Перезагрузите компьютер

✅ ПРОВЕРКА УСТАНОВКИ:

   1. Откройте командную строку (Win+R → cmd → Enter)
   2. Введите: dotnet --version
   3. Должна отобразиться версия: 6.0.xxx
   4. Если версия отображается - установка успешна!

⚠️ ВАЖНО: Устанавливайте именно .NET SDK, а не .NET Runtime!
   SDK включает в себя всё необходимое для сборки и запуска приложения.

2.5  РЕШЕНИЕ ПРОБЛЕМ ПРИ ЗАПУСКЕ
───────────────────────────────────────────────────────────────────────────────

❌ ПРОБЛЕМА: "dotnet не является внутренней или внешней командой"
───────────────────────────────────────────────────────────────────────────────

   Причина: Не установлен .NET SDK или не добавлен в PATH
   
   Решение:
   1. Установите .NET SDK 6.0 (см. раздел 2.2)
   2. Перезагрузите компьютер
   3. Проверьте: dotnet --version
   4. Если не помогло - переустановите .NET SDK

❌ ПРОБЛЕМА: "Build FAILED" при первом запуске
───────────────────────────────────────────────────────────────────────────────

   Причина: Нет подключения к интернету (нужно для загрузки NuGet пакетов)
   
   Решение:
   1. Проверьте подключение к интернету
   2. Запустите FINAL_BUILD_AND_RUN.bat
   3. Дождитесь завершения сборки

❌ ПРОБЛЕМА: Приложение не запускается
───────────────────────────────────────────────────────────────────────────────

   Причина: Ошибка при запуске .exe файла
   
   Решение:
   1. Запустите START.bat от имени администратора
   2. Если не помогло - запустите FINAL_BUILD_AND_RUN.bat

💡 СОВЕТ: Если ничего не помогает, попробуйте:
   1. Удалить папки bin и obj в DatabaseApp
   2. Запустить FINAL_BUILD_AND_RUN.bat
   3. Если и это не помогло - переустановите .NET SDK

4.2  СВЯЗИ МЕЖДУ ТАБЛИЦАМИ
───────────────────────────────────────────────────────────────────────────────

🔗 FOREIGN KEY CONSTRAINTS:

   1. customer_orders → customers
   ─────────────────────────────────────────────────────────────────────────
   customer_orders.customer_id → customers.customer_id
   
   Связь: Каждый заказ принадлежит одному заказчику
   Тип: Many-to-One (много заказов к одному заказчику)
   Действие при удалении: Не указано (по умолчанию RESTRICT)

   2. order_items → customer_orders
   ─────────────────────────────────────────────────────────────────────────
   order_items.order_id → customer_orders.order_id ON DELETE CASCADE
   
   Связь: Каждая позиция принадлежит одному заказу
   Тип: Many-to-One (много позиций к одному заказу)
   Действие при удалении: CASCADE (при удалении заказа удаляются все позиции)

   3. production_materials → production
   ─────────────────────────────────────────────────────────────────────────
   production_materials.production_id → production.production_id ON DELETE CASCADE
   
   Связь: Каждый материал принадлежит одному производству
   Тип: Many-to-One (много материалов к одному производству)
   Действие при удалении: CASCADE (при удалении производства удаляются все материалы)

   4. specification_materials → specifications
   ─────────────────────────────────────────────────────────────────────────
   specification_materials.spec_id → specifications.spec_id ON DELETE CASCADE
   
   Связь: Каждый материал принадлежит одной спецификации
   Тип: Many-to-One (много материалов к одной спецификации)
   Действие при удалении: CASCADE (при удалении спецификации удаляются все материалы)

   5. documents → counterparties (ЕСЛИ ЕСТЬ)
   ─────────────────────────────────────────────────────────────────────────
   documents.counterparty_id → counterparties.counterparty_id
   
   Связь: Каждый документ связан с одним контрагентом
   Тип: Many-to-One
   Действие при удалении: Не указано

   6. transactions → documents (ЕСЛИ ЕСТЬ)
   ─────────────────────────────────────────────────────────────────────────
   transactions.document_id → documents.document_id ON DELETE CASCADE
   
   Связь: Каждая транзакция связана с одним документом
   Тип: Many-to-One
   Действие при удалении: CASCADE

   6. Запускает приложение
   7. Показывает понятные ошибки

   Использование:

  ПРИЛОЖЕНИЕ A: ЧАСТО ЗАДАВАЕМЫЕ ВОПРОСЫ (FAQ)
═══════════════════════════════════════════════════════════════════════════════

❓ Как запустить приложение?
───────────────────────────────────────────────────────────────────────────────

   Самый простой способ - двойной клик на START.bat
   Приложение запустится автоматически.

❓ Что делать, если приложение не запускается?
───────────────────────────────────────────────────────────────────────────────

   1. Проверьте наличие .NET SDK 6.0: dotnet --version
   2. Если нет - установите .NET SDK (см. раздел 2.2)
   3. Запустите FINAL_BUILD_AND_RUN.bat
   4. Если не помогло - запустите от имени администратора

❓ Где находится база данных?
───────────────────────────────────────────────────────────────────────────────

   DatabaseApp\bin\Debug\net6.0-windows\accounting.db
   
   База данных создаётся автоматически при первом запуске.

❓ Как добавить новые данные?
───────────────────────────────────────────────────────────────────────────────

   1. Войдите как admin (admin/admin123)
   2. Откройте нужный раздел (например, "Заказчики")
   3. Нажмите кнопку "Добавить"
   4. Заполните поля
   5. Нажмите "Сохранить"

❓ Можно ли удалить базу данных и создать заново?
───────────────────────────────────────────────────────────────────────────────

   Да. Удалите файл accounting.db и запустите приложение.
   База данных будет создана заново с начальными данными.

❓ Как изменить учётные данные?
───────────────────────────────────────────────────────────────────────────────

   Учётные данные хранятся в таблице users.
   Можно изменить через SQL запрос или в коде DatabaseHelper.cs

❓ Почему таблица counterparties не используется?
───────────────────────────────────────────────────────────────────────────────

   Таблица создана, но в приложении используется таблица customers.
   Это сделано для совместимости с импортированными данными.

❓ Как экспортировать данные из базы?
───────────────────────────────────────────────────────────────────────────────

   Можно использовать SQLite Browser или написать код экспорта в Excel.
   Файл accounting.db можно открыть в любом SQLite клиенте.

❓ Можно ли использовать другую базу данных (MySQL, PostgreSQL)?
───────────────────────────────────────────────────────────────────────────────

   Да, но потребуется изменить DatabaseHelper.cs и заменить
   System.Data.SQLite на соответствующий провайдер.

❓ Как добавить новое окно в приложение?
───────────────────────────────────────────────────────────────────────────────

   1. Создайте новый XAML файл в папке Views
   2. Создайте соответствующий .cs файл
   3. Добавьте кнопку в MainWindow.xaml
   4. Добавьте обработчик клика в MainWindow.xaml.cs

  ПРИЛОЖЕНИЕ B: РЕШЕНИЕ ТИПИЧНЫХ ПРОБЛЕМ
═══════════════════════════════════════════════════════════════════════════════

❌ ПРОБЛЕМА: "dotnet не является внутренней или внешней командой"
───────────────────────────────────────────────────────────────────────────────

   ПРИЧИНА: Не установлен .NET SDK или не добавлен в PATH
   
   РЕШЕНИЕ:
   1. Скачайте .NET SDK 6.0 с https://dotnet.microsoft.com/download/dotnet/6.0
   2. Установите SDK
   3. Перезагрузите компьютер
   4. Проверьте: dotnet --version

❌ ПРОБЛЕМА: "Build FAILED" при первом запуске
───────────────────────────────────────────────────────────────────────────────

   ПРИЧИНА: Нет подключения к интернету (нужно для загрузки NuGet пакетов)
   
   РЕШЕНИЕ:
   1. Проверьте подключение к интернету
   2. Запустите FINAL_BUILD_AND_RUN.bat
   3. Дождитесь завершения сборки

❌ ПРОБЛЕМА: "Unable to open database file"
───────────────────────────────────────────────────────────────────────────────

   ПРИЧИНА: Нет прав доступа к папке или антивирус блокирует
   
   РЕШЕНИЕ:
   1. Запустите от имени администратора
   2. Добавьте папку database_dem в исключения антивируса
   3. Проверьте свободное место на диске

❌ ПРОБЛЕМА: Приложение запускается, но окна не открываются
───────────────────────────────────────────────────────────────────────────────

   ПРИЧИНА: Ошибка в коде окна или отсутствует файл
   
   РЕШЕНИЕ:
   1. Проверьте наличие всех файлов в папке Views
   2. Проверьте консоль на наличие ошибок
   3. Запустите через Visual Studio для отладки

❌ ПРОБЛЕМА: Данные не отображаются в таблицах
───────────────────────────────────────────────────────────────────────────────

   ПРИЧИНА: База данных не создана или пуста
   
   РЕШЕНИЕ:
   1. Проверьте наличие файла accounting.db
   2. Удалите файл и запустите приложение заново
   3. Проверьте метод InsertInitialData() в DatabaseHelper.cs

❌ ПРОБЛЕМА: Медленный запуск (более 5 минут)
───────────────────────────────────────────────────────────────────────────────

   ПРИЧИНА: Слабый компьютер или антивирус сканирует файлы
   
   РЕШЕНИЕ:
   1. Закройте лишние программы
   2. Добавьте папку database_dem в исключения антивируса
   3. Дождитесь первой сборки (она самая долгая)

❌ ПРОБЛЕМА: "Неверный логин или пароль" при входе
───────────────────────────────────────────────────────────────────────────────

   ПРИЧИНА: Неправильные учётные данные или таблица users пуста
   
   РЕШЕНИЕ:
   1. Проверьте учётные данные: admin/admin123 или user/user123
   2. Проверьте раскладку клавиатуры (должна быть английская)
   3. Удалите базу данных и создайте заново

❌ ПРОБЛЕМА: Ошибка "System.Data.SQLite.dll not found"
───────────────────────────────────────────────────────────────────────────────

   ПРИЧИНА: Не установлен NuGet пакет System.Data.SQLite
   
   РЕШЕНИЕ:
   1. Запустите: dotnet restore
   2. Запустите: dotnet build
   3. Если не помогло - переустановите пакет через NuGet

❌ ПРОБЛЕМА: Приложение закрывается сразу после запуска
───────────────────────────────────────────────────────────────────────────────

   ПРИЧИНА: Критическая ошибка при инициализации
   
   РЕШЕНИЕ:
   1. Запустите через командную строку: DatabaseApp.exe
   2. Посмотрите текст ошибки
   3. Запустите через Visual Studio для отладки

❌ ПРОБЛЕМА: Не могу найти файл START.bat
───────────────────────────────────────────────────────────────────────────────

   ПРИЧИНА: Файл не создан или удалён
   
   РЕШЕНИЕ:
   1. Создайте файл START.bat вручную (см. раздел 6.1)
   2. Или запустите через командную строку: dotnet run

ЧАСТЬ 3: СОЗДАНИЕ ТАБЛИЦ (CreateTables)

        private static void CreateTables()
        {
            using (var connection = GetConnection())
            {
                connection.Open();

                string[] createTableQueries = new string[]
                {
                    // Таблица пользователей
                    @"CREATE TABLE IF NOT EXISTS users (
                        user_id INTEGER PRIMARY KEY AUTOINCREMENT,
                        username TEXT NOT NULL UNIQUE,
                        password_hash TEXT NOT NULL,
                        role TEXT NOT NULL DEFAULT 'user',
                        full_name TEXT,
                        created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
                        last_login DATETIME
                    )",

                    // Таблица заказчиков
                    @"CREATE TABLE IF NOT EXISTS customers (
                        customer_id TEXT PRIMARY KEY,
                        name TEXT NOT NULL,
                        inn TEXT,
                        address TEXT,
                        phone TEXT,
                        is_salesman INTEGER DEFAULT 0,
                        is_buyer INTEGER DEFAULT 0,
                        created_at DATETIME DEFAULT CURRENT_TIMESTAMP
                    )",

                    // ... остальные таблицы ...
                };

                using (var command = connection.CreateCommand())
                {
                    foreach (var query in createTableQueries)
                    {
                        command.CommandText = query;
                        command.ExecuteNonQuery();
                    }
                }
            }
        }

📝 ОБЪЯСНЕНИЕ:

   using (var connection = GetConnection())
   ─────────────────────────────────────────────────────────────────────────
   • using - автоматически закрывает соединение после использования
   • var - компилятор сам определит тип (SQLiteConnection)
   • GetConnection() - метод, который создаёт соединение с БД

   connection.Open();
   ─────────────────────────────────────────────────────────────────────────
   Открыть соединение с базой данных
   После этого можно выполнять SQL запросы

   string[] createTableQueries = new string[] { ... }
   ─────────────────────────────────────────────────────────────────────────
   Массив строк с SQL запросами
   Каждый элемент массива - это SQL запрос CREATE TABLE

   @"CREATE TABLE IF NOT EXISTS users (...)"
   ─────────────────────────────────────────────────────────────────────────
   • @ - вербатим строка (можно писать на нескольких строках)
   • CREATE TABLE - SQL команда создания таблицы
   • IF NOT EXISTS - создать только если таблицы ещё нет
   • users - имя таблицы

   СТРУКТУРА ТАБЛИЦЫ users:
   ─────────────────────────────────────────────────────────────────────────
   
   user_id INTEGER PRIMARY KEY AUTOINCREMENT
   • user_id - имя поля (столбца)
   • INTEGER - тип данных (целое число)
   • PRIMARY KEY - первичный ключ (уникальный идентификатор)
   • AUTOINCREMENT - автоматически увеличивается (1, 2, 3, ...)

   username TEXT NOT NULL UNIQUE
   • username - логин пользователя
   • TEXT - тип данных (текст)
   • NOT NULL - поле обязательно для заполнения
   • UNIQUE - значение должно быть уникальным

   password_hash TEXT NOT NULL
   • password_hash - пароль пользователя
   • NOT NULL - обязательное поле

   role TEXT NOT NULL DEFAULT 'user'
   • role - роль пользователя (admin или user)
   • DEFAULT 'user' - значение по умолчанию

   full_name TEXT
   • full_name - полное имя пользователя
   • Может быть пустым (нет NOT NULL)

   created_at DATETIME DEFAULT CURRENT_TIMESTAMP
   • created_at - дата создания записи
   • DATETIME - тип данных (дата и время)
   • DEFAULT CURRENT_TIMESTAMP - автоматически текущая дата/время

   last_login DATETIME
   • last_login - дата последнего входа
   • Может быть пустым

   СТРУКТУРА ТАБЛИЦЫ customers:
   ─────────────────────────────────────────────────────────────────────────
   
   customer_id TEXT PRIMARY KEY
   • Уникальный идентификатор заказчика (например: "000000001")
   • TEXT - потому что ID содержит ведущие нули

   name TEXT NOT NULL
   • Название организации (например: ООО "Ассоль")

   inn TEXT
   • ИНН организации (может быть пустым)

   address TEXT
   • Адрес организации

   phone TEXT
   • Телефон организации

   is_salesman INTEGER DEFAULT 0
   • Является ли продавцом (0 = нет, 1 = да)
   • INTEGER используется как boolean

   is_buyer INTEGER DEFAULT 0
   • Является ли покупателем (0 = нет, 1 = да)

   foreach (var query in createTableQueries)
   ─────────────────────────────────────────────────────────────────────────
   Цикл по всем SQL запросам в массиве
   Выполняет каждый запрос CREATE TABLE

   command.CommandText = query;
   ─────────────────────────────────────────────────────────────────────────
   Устанавливает текст SQL команды

   command.ExecuteNonQuery();
   ─────────────────────────────────────────────────────────────────────────
   Выполняет SQL команду

ЧАСТЬ 5: МЕТОДЫ ДЛЯ РАБОТЫ С БАЗОЙ ДАННЫХ
───────────────────────────────────────────────────────────────────────────────

        public static SQLiteConnection GetConnection()
        {
            return new SQLiteConnection(connectionString);
        }

        public static DataTable ExecuteQuery(string query)
        {
            using (var connection = GetConnection())
            {
                connection.Open();
                using (var command = new SQLiteCommand(query, connection))
                {
                    using (var adapter = new SQLiteDataAdapter(command))
                    {
                        DataTable dataTable = new DataTable();
                        adapter.Fill(dataTable);
                        return dataTable;
                    }
                }
            }
        }

        public static int ExecuteNonQuery(string query)
        {
            using (var connection = GetConnection())
            {
                connection.Open();
                using (var command = new SQLiteCommand(query, connection))
                {
                    return command.ExecuteNonQuery();
                }
            }
        }

        public static object ExecuteScalar(string query)
        {
            using (var connection = GetConnection())
            {
                connection.Open();
                using (var command = new SQLiteCommand(query, connection))
                {
                    return command.ExecuteScalar();
                }
            }
        }

        public static bool ValidateUser(string username, string password, out string? role)
        {
            role = null;
            string query = $"SELECT role FROM users WHERE username = '{username}' AND password_hash = '{password}'";
            
            using (var connection = GetConnection())
            {
                connection.Open();
                using (var command = new SQLiteCommand(query, connection))
                {
                    var result = command.ExecuteScalar();
                    if (result != null)
                    {
                        role = result.ToString() ?? "";
                        
                        // Обновляем время последнего входа
                        string updateQuery = $"UPDATE users SET last_login = datetime('now') WHERE username = '{username}'";
                        ExecuteNonQuery(updateQuery);
                        
                        return true;
                    }
                }
            }
            
            return false;
        }

💡 ИТОГО: DatabaseHelper.cs предоставляет 5 основных методов:
   1. GetConnection() - создание соединения
   2. ExecuteQuery() - выполнение SELECT запросов
   3. ExecuteNonQuery() - выполнение INSERT/UPDATE/DELETE
   4. ExecuteScalar() - получение одного значения

        {

            string password = PasswordBox.Password;

            if (string.IsNullOrEmpty(username) || string.IsNullOrEmpty(password))
            {
                MessageBox.Show("Введите логин и пароль!", "Ошибка", 
                    MessageBoxButton.OK, MessageBoxImage.Warning);
                return;
            }

            if (DatabaseHelper.ValidateUser(username, password, out string role))
            {
                MainWindow mainWindow = new MainWindow(username, role);
                mainWindow.Show();
                this.Close();
            }
            else
            {
                MessageBox.Show("Неверный логин или пароль!", "Ошибка", 
                    MessageBoxButton.OK, MessageBoxImage.Error);
            }
        }
    }
}

private void LoadCustomers()
{
    string query = "SELECT * FROM customers ORDER BY name";
    DataTable dt = DatabaseHelper.ExecuteQuery(query);
    CustomersDataGrid.ItemsSource = dt.DefaultView;
}

private void AddButton_Click(object sender, RoutedEventArgs e)
{
    AddEditCustomerWindow window = new AddEditCustomerWindow();
    if (window.ShowDialog() == true)
    {
        LoadCustomers();
    }
}

private void DeleteButton_Click(object sender, RoutedEventArgs e)
{
    if (CustomersDataGrid.SelectedItem == null)
    {
        MessageBox.Show("Выберите заказчика для удаления!");
        return;
    }

    var result = MessageBox.Show("Удалить выбранного заказчика?", 
        "Подтверждение", MessageBoxButton.YesNo, MessageBoxImage.Question);

    if (result == MessageBoxResult.Yes)
    {
        DataRowView row = (DataRowView)CustomersDataGrid.SelectedItem;
        string customerId = row["customer_id"].ToString();

        string query = $"DELETE FROM customers WHERE customer_id = '{customerId}'";
        DatabaseHelper.ExecuteNonQuery(query);

        LoadCustomers();
        MessageBox.Show("Заказчик удалён!");
    }
}

ШАГ 13: СОЗДАНИЕ BAT ФАЙЛОВ ДЛЯ ЗАПУСКА
───────────────────────────────────────────────────────────────────────────────

Создайте файл START.bat в корневой папке проекта:

@echo off
chcp 65001 > nul
echo ═══════════════════════════════════════════════════════════════
echo   ЗАПУСК ПРИЛОЖЕНИЯ
echo ═══════════════════════════════════════════════════════════════
echo.

echo Проверка .NET SDK...
dotnet --version
if errorlevel 1 (
    echo ОШИБКА: .NET SDK не установлен!
    echo Установите .NET 6.0 SDK с https://dotnet.microsoft.com/download/dotnet/6.0
    pause
    exit /b 1
)

echo.
echo Поиск готового приложения...
set EXE_PATH=DatabaseApp\bin\Debug\net6.0-windows\DatabaseApp.exe

if exist "%EXE_PATH%" (
    echo Найден готовый файл. Запуск...
    start "" "%EXE_PATH%"
) else (
    echo Готовый файл не найден. Сборка проекта...
    cd DatabaseApp
    dotnet build -c Debug
    if errorlevel 1 (
        echo ОШИБКА: Сборка не удалась!
        pause
        exit /b 1
    )
    cd ..
    echo Запуск приложения...
    start "" "%EXE_PATH%"
)

echo.
echo Приложение запущено!
exit

✅ Результат: Создан BAT файл для быстрого запуска
