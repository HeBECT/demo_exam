1.1  ПРИВЕТСТВИЕ И ОБЗОР ПРОЕКТА
───────────────────────────────────────────────────────────────────────────────

🎓 ДОБРО ПОЖАЛОВАТЬ!

Система бухгалтерского учета - полнофункциональное WPF приложение для 
демонстрации навыков работы с базами данных на экзамене БОМ 26.

✨ КЛЮЧЕВЫЕ ВОЗМОЖНОСТИ:
  • Работа с базой данных SQLite (автономная, без сервера)
  • CRUD операции для справочников (добавление, редактирование, удаление)
  • Просмотр документов и отчётов
  • Система авторизации (admin/user)
  • Прокручиваемое меню (колёсиком мыши)
  • Автоматическое создание БД при первом запуске
  • Импорт данных из Excel файлов

🛠️ ТЕХНОЛОГИИ:
  • Язык: C# 10
  • Платформа: .NET 6.0
  • Интерфейс: WPF (Windows Presentation Foundation)
  • База данных: SQLite 3
  • ORM: ADO.NET

📊 СТАТИСТИКА ПРОЕКТА:
  • Таблиц в БД: 9
  • Записей: 29
  • Окон: 9
  • Моделей: 10
  • Строк кода: ~3000+
  • Документации: 22 файла
  • BAT файлов: 6

   1. Запустите скачанный файл

💡 Если версия отображается - установка успешна!

2.5  РЕШЕНИЕ ПРОБЛЕМ
───────────────────────────────────────────────────────────────────────────────

❌ ПРОБЛЕМА 1: Приложение не запускается

🔍 Причины:
  • Не установлен .NET SDK
  • Неправильная версия .NET
  • Повреждены файлы проекта

✅ Решение:
  1. Проверьте: dotnet --version (должна быть 6.0+)
  2. Если нет - установите .NET 6.0 SDK
  3. Перезагрузите компьютер
  4. Запустите START.bat заново

❌ ПРОБЛЕМА 2: "dotnet не является командой"

✅ Решение:
  1. Установите .NET 6.0 SDK
  2. Перезагрузите компьютер
  3. Проверьте: dotnet --version

❌ ПРОБЛЕМА 3: Ошибка компиляции

✅ Решение:
  1. Проверьте интернет (нужен при первом запуске)
  2. Запустите START.bat заново
  3. Если не помогло - запустите ПОЛНАЯ_ПЕРЕСБОРКА.bat

❌ ПРОБЛЕМА 4: База данных не создаётся

✅ Решение:
  1. Запустите START.bat от имени администратора
  2. Проверьте свободное место (нужно 100+ МБ)
  3. Добавьте папку в исключения антивируса

💡 Это самый важный файл проекта! Он отвечает за:

📍 Расположение: DatabaseApp\DatabaseHelper.cs
📏 Размер: ~450 строк кода

🔑 КЛЮЧЕВЫЕ МЕТОДЫ:

   InitializeDatabase()
   • Проверяет существование БД
   • Создаёт файл accounting.db
   • Вызывает CreateTables()
   • Вызывает InsertInitialData()
   
   CreateTables()
   • Создаёт 9 таблиц
   • Устанавливает связи (FOREIGN KEY)
   • Добавляет индексы

   • Вставляет 2 пользователей

   • Возвращает подключение к БД

   ValidateUser(string username, string password)

       if (string.IsNullOrEmpty(username) || string.IsNullOrEmpty(password))

           MessageBox.Show("Введите логин и пароль!");
           return;

           this.Close();

ПРИЛОЖЕНИЕ A: ЧАСТО ЗАДАВАЕМЫЕ ВОПРОСЫ (FAQ)
───────────────────────────────────────────────────────────────────────────────

❓ В: Какая база данных используется?
✅ О: SQLite - автономная встроенная база данных.
      Не требует установки MySQL, PostgreSQL или других серверов.
      Файл: accounting.db в папке bin\Debug\net6.0-windows\

❓ В: Где хранятся данные?
✅ О: Все данные хранятся в одном файле accounting.db
      Расположение: DatabaseApp\bin\Debug\net6.0-windows\accounting.db
      Размер: ~100 КБ

❓ В: Как сбросить базу данных?
✅ О: 1. Закройте приложение
      2. Удалите файл accounting.db
      3. Запустите приложение заново
      4. База будет создана с начальными данными

❓ В: Откуда взялись данные в БД?
✅ О: Данные импортированы из 6 Excel/JSON файлов:
      • Заказчики.json - 6 заказчиков
      • Цены.xlsx - 8 позиций продукции
      • Заказ покупателя.xlsx - 1 заказ с 3 позициями
      • Производство.xlsx - 1 производство с 2 материалами
      • Спецификация.xlsx - 1 спецификация с 2 материалами
      • Расчет стоимости.xlsx - 3 расчёта

❓ В: Сколько таблиц в базе данных?
✅ О: 9 таблиц с полной структурой и связями:
      1. users (пользователи)
      2. customers (заказчики)
      3. products (продукция и материалы)
      4. customer_orders (заказы покупателей)
      5. order_items (позиции заказов)
      6. production (производство)
      7. production_materials (материалы производства)
      8. specifications (спецификации)
      9. specification_materials (материалы спецификаций)
      10. cost_calculations (расчёт стоимости)

❓ В: Какие операции поддерживаются?
✅ О: • CRUD (Create, Read, Update, Delete) для справочников
      • Просмотр документов (заказы, производство)
      • Просмотр отчётов (расчёт стоимости, спецификации)
      • Авторизация пользователей (admin/user)

❓ В: Можно ли добавить новые данные?
✅ О: Да! В справочниках "Заказчики" и "Продукция и материалы"
      есть полный функционал добавления, редактирования и удаления.

❓ В: Как реализована авторизация?
✅ О: Простая авторизация с проверкой логина и пароля в таблице users.
      Два пользователя:
      • admin/admin123 (полный доступ)
      • user/user123 (просмотр)
      Пароли хранятся в открытом виде для демонстрации.

❓ В: Почему левое меню прокручивается?
✅ О: Чтобы все пункты меню были доступны даже на маленьких экранах.
      Прокрутка работает колёсиком мыши - это удобно!

❓ В: Нужен ли интернет для работы?
✅ О: Интернет нужен только при первом запуске для загрузки NuGet пакетов.
      После первой сборки приложение работает полностью автономно.

❓ В: Можно ли запустить на другом компьютере?
✅ О: Да! Скопируйте всю папку database_dem на другой компьютер.
      Требования: Windows 7+ и .NET 6.0 SDK.

❓ В: Как открыть проект в Visual Studio?
✅ О: 1. Откройте Visual Studio 2022
      2. Нажмите "Open a project or solution"
      3. Найдите файл DatabaseApp.csproj
      4. Нажмите "Open"

❓ В: Можно ли изменить учётные данные?
✅ О: Да! Откройте файл accounting.db в SQLite браузере
      и измените таблицу users. Или измените код в DatabaseHelper.cs
      в методе InsertInitialData().

❓ В: Какие технологии использованы?
✅ О: • C# 10
      • .NET 6.0
      • WPF (Windows Presentation Foundation)
      • SQLite 3
      • ADO.NET

❓ В: Есть ли документация?
✅ О: Да! Полная документация:
      • ПОЛНЫЙ_ГАЙД_УЛУЧШЕННЫЙ.txt (этот файл)
      • ПОЛНЫЙ_ГАЙД_ДЛЯ_WORD.txt (оригинальная версия)
      • README.md (краткое описание)
      • Полный гайд.docx (версия для Word)

❓ В: Сколько времени занимает запуск?
✅ О: • Первый раз: 1-3 минуты (автоматическая сборка)
      • Последующие: 1-5 секунд (готовый файл)

❓ В: Можно ли экспортировать данные?
✅ О: В текущей версии нет функции экспорта.
      Но можно открыть accounting.db в SQLite браузере
      и экспортировать данные в CSV/Excel.

✅ О: Удалите файл accounting.db и запустите приложение заново.
